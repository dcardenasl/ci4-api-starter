# Notas de Version — 4 de marzo de 2026

## Resumen
- se fortalecio el pipeline de request con `RequestDataCollector` + `RequestDtoFactory` para que los controladores nunca llamen facades estaticos y todos los DTO compartan la misma instancia de validador antes de ejecutar la logica de negocio.
- se reforzo auditoria/logging inyectando `AuditServiceInterface` en cada modelo que usa el trait `Auditable`, sanitizando tokens a nivel de entidad y agregando tests de proteccion contra facades estaticos. `UserEntity::toArray()` ahora elimina password, tokens y campos de verificacion de forma explicita.
- se simplificaron repositorios y servicios de archivos (un solo entrypoint `destroy`, repositorio base sin estado) manteniendo la documentacion y cobertura de tests actualizadas.
- se empaquetaron todas las recomendaciones de auditoria, refactors DTO y pruebas en este release, con notas en `README`, docs de arquitectura y reporte de auditoria.

## Pruebas
- `php spark tests:prepare-db && vendor/bin/phpunit --colors=never` (541 tests / 1212 assertions, 2 skipped, solo el warning estandar “No code coverage driver available”).

## Notas de Migracion
- Reemplazar cada `new SomeRequestDTO(...)` por `new SomeRequestDTO(..., service('validation'))` cuando se instancien DTOs manualmente (tests o flujos internos) hasta agregar wrappers dedicados.
- Asegurar que cualquier modelo nuevo que use `Auditable` llame `setAuditService(Services::auditService())` antes de `initAuditable()`.
