# Release Notes — March 4, 2026

## Overview
- tightened the request pipeline with `RequestDataCollector` + `RequestDtoFactory` so controllers never call static facades and all DTOs share the same validator instance before business logic executes.
- hardened audit/logging by injecting `AuditServiceInterface` into each model that uses the `Auditable` trait, sanitizing tokens at the entity level, and adding static-facade guard tests. `UserEntity::toArray()` now strips passwords, tokens, and verification fields explicitly.
- simplified repositories and file services (single `destroy` entrypoint, stateless base repository) while keeping the documentation/test coverage current.
- packaged all audit recommendations, DTO refactors, and tests into this release and noted them in `README`, architecture docs, and the audit report.
- introduced hybrid audit delivery: critical security events stay synchronous (best effort), while non-critical events are enqueued in the new `audit` queue with safe fallback to synchronous writes.

## Testing
- `php spark tests:prepare-db && vendor/bin/phpunit --colors=never` (541 tests / 1212 assertions, 2 skipped, only the standard “No code coverage driver available” warning).

## Migration Notes
- Replace every `new SomeRequestDTO(...)` with `new SomeRequestDTO(..., service('validation'))` when instantiating DTOs manually (tests or internal flows) until we add helper wrappers.
- Ensure any new model using `Auditable` calls `setAuditService(Services::auditService())` before `initAuditable()`.
- Configure and operate audit queue workers:
  - `AUDIT_ASYNC_ENABLED=true|false`
  - `AUDIT_QUEUE_NAME=audit`
  - `AUDIT_MAX_PAYLOAD_BYTES=60000`
  - worker example: `php spark queue:work --queue=audit`
